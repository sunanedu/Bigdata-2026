## 🔍 ทำไมคุณภาพข้อมูลถึงสำคัญ?

ลองนึกถึงสถานการณ์นี้:

> 🏥 **เหตุการณ์จริงในโรงพยาบาลแห่งหนึ่ง**
> แพทย์สั่งยาให้ผู้ป่วยโดยอ้างอิงจากน้ำหนักในระบบ — **แต่ข้อมูลน้ำหนักผิดพลาด** บันทึกเป็น 550 กก. แทนที่จะเป็น 55 กก. เพราะพิมพ์เลขผิดช่อง
> ผลลัพธ์: ระบบคำนวณขนาดยาเกินจริง 10 เท่า → อาจเป็นอันตรายถึงชีวิต

ข้อมูลที่ผิดพลาดเพียงตัวเดียว → การตัดสินใจที่ผิดพลาด → ผลกระทบมหาศาล

> **กฎทอง (Golden Rule) ของข้อมูล:**
> **"Garbage In, Garbage Out"** (กาบิจ อิน, กาบิจ เอาท์)
> = ข้อมูลขยะเข้า → คำตอบขยะออก
> ไม่ว่าจะมีระบบวิเคราะห์ดีแค่ไหน ถ้าข้อมูลตั้งต้นผิด ผลลัพธ์ก็ผิดเสมอ

## 📐 6 มิติของคุณภาพข้อมูล (6 Dimensions of Data Quality)

**Data Quality** (เดต้า ควอลิตี้) หมายถึง ระดับความเหมาะสมของข้อมูลที่จะนำไปใช้งานได้จริง โดยวัดจาก 6 มิติหลัก:

---

### มิติที่ 1 — Accuracy (แอคคิวเรซี่) — "ความถูกต้อง"

**ความหมาย:** ข้อมูลตรงกับความเป็นจริงหรือไม่?

> **เปรียบเทียบ:** เหมือนนาฬิกาที่ต้องตรงกับเวลาจริง ถ้าเร็วหรือช้า = ไม่ถูกต้อง

**ตัวอย่างจากข้อมูลอุบัติเหตุ:**
- ✅ ถูกต้อง: `อายุ_ปี = 24` → ผู้ขับขี่อายุ 24 ปีจริง
- ❌ ไม่ถูกต้อง: `อายุ_ปี = 240` → พิมพ์เลขเกินหนึ่งตัว กลายเป็น 240 ปี (เป็นไปไม่ได้)
- ❌ ไม่ถูกต้อง: `ความเร็วโดยประมาณ_กมชม = 500` → รถในประเทศไทยไม่มีทางวิ่ง 500 กม./ชม.

**วิธีตรวจสอบ:**
```
ถามตัวเองว่า → "ค่านี้เป็นไปได้จริงในโลกความเป็นจริงไหม?"
```

---

### มิติที่ 2 — Completeness (คอมพลีทเนส) — "ความสมบูรณ์"

**ความหมาย:** ข้อมูลมีครบทุกช่องที่ควรมีหรือไม่? ไม่มีค่าว่าง (NULL) ในจุดสำคัญ

> **เปรียบเทียบ:** เหมือนแบบฟอร์มสมัครงานที่ต้องกรอกทุกช่อง ถ้าเว้นว่างช่อง "ชื่อ" ไว้ → แบบฟอร์มไม่สมบูรณ์

**ตัวอย่างจากข้อมูลอุบัติเหตุ:**
- ✅ สมบูรณ์: ทุก record มี `จังหวัด` ระบุครบ
- ❌ ไม่สมบูรณ์: บาง record ไม่มี `หมายเลขถนน` (NULL) — แต่ก็ยอมรับได้เพราะถนนในหมู่บ้านอาจไม่มีหมายเลข
- ❌ ไม่สมบูรณ์ (วิกฤต): ไม่มี `วันที่เกิดเหตุ` — วิเคราะห์แนวโน้มรายเดือนไม่ได้เลย

**วิธีตรวจสอบ:**
```
นับ NULL แต่ละคอลัมน์ → ถ้า > 5% ต้องพิจารณาแก้ไข
```

---

### มิติที่ 3 — Consistency (คอนซิสเทนซี่) — "ความสอดคล้อง"

**ความหมาย:** ข้อมูลในจุดต่าง ๆ ตรงกันไหม? ไม่มีความขัดแย้งในตัวเอง

> **เปรียบเทียบ:** เหมือนคนที่บอกว่าเกิดปี 2545 แต่ระบุอายุว่า 50 ปี — ไม่สอดคล้องกัน!

**ตัวอย่างจากข้อมูลอุบัติเหตุ:**
- ❌ ไม่สอดคล้อง: `ความเร็วโดยประมาณ = 30 กม./ชม.` แต่ `เกินความเร็ว_ใช่ไม่ใช่ = ใช่` (จำกัดที่ 60 กม./ชม.) → ขัดแย้งกัน!
- ❌ ไม่สอดคล้อง: `จำนวนผู้เสียชีวิต = 3` แต่ `ความรุนแรง = ไม่บาดเจ็บ` → ตายแล้วไม่บาดเจ็บ?
- ✅ สอดคล้อง: `จำนวนผู้เสียชีวิต = 1` และ `ความรุนแรง = เสียชีวิต` → ตรงกัน

**วิธีตรวจสอบ:**
```
ตรวจสอบความสัมพันธ์ระหว่างคอลัมน์ว่า "ค่าเหล่านี้ควรจะเป็นไปด้วยกันได้ไหม?"
```

---

### มิติที่ 4 — Timeliness (ไทม์ลีเนส) — "ความทันสมัย"

**ความหมาย:** ข้อมูลทันสมัย เป็นปัจจุบัน และพร้อมใช้งานในเวลาที่ต้องการ

> **เปรียบเทียบ:** ข่าวสารที่ช้าไป 1 สัปดาห์ = ไม่มีประโยชน์สำหรับการตัดสินใจวันนี้

**ตัวอย่างจากข้อมูลอุบัติเหตุ:**
- ✅ ทันสมัย: ข้อมูลอัปเดตทุกวัน สามารถดูสถิติย้อนหลัง 24 ชั่วโมงได้
- ❌ ล้าสมัย: มีข้อมูลเดือน มกราคม–มิถุนายน แต่ตอนนี้เดือนธันวาคมแล้ว — ข้อมูลครึ่งปีหลังหายไปไหน?
- ❌ ล้าสมัย: ข้อมูลประชากรจังหวัดใช้ตัวเลขปี 2560 ทั้งที่ปัจจุบันเป็นปี 2568

**วิธีตรวจสอบ:**
```
ดู timestamp ล่าสุดของข้อมูล → เก่าเกินไปไหม?
```

---

### มิติที่ 5 — Uniqueness (ยูนีคเนส) — "ความเป็นเอกลักษณ์"

**ความหมาย:** ไม่มีข้อมูลซ้ำซ้อน (Duplicate) ที่ทำให้นับผิด

> **เปรียบเทียบ:** ถ้าบันทึกชื่อนักเรียนคนเดียวซ้ำ 3 ครั้ง → นับจำนวนนักเรียนผิด!

**ตัวอย่างจากข้อมูลอุบัติเหตุ:**
- ✅ ไม่ซ้ำ: แต่ละ `accident_id` ไม่ซ้ำกัน (ACC2568000001, ACC2568000002, ...)
- ❌ ซ้ำ: มี record ที่มี `accident_id` เดิม, `วันที่` เดิม, `สถานที่` เดิม ปรากฏ 2 ครั้ง = บันทึกซ้ำ
- ❌ ซ้ำ (แฝง): `จังหวัด = "กรุงเทพ"` และ `จังหวัด = "กรุงเทพฯ"` → ระบบนับเป็น 2 จังหวัดต่างกัน!

**วิธีตรวจสอบ:**
```
COUNT(*) vs COUNT(DISTINCT primary_key) → ถ้าต่างกัน = มีซ้ำ
```

---

### มิติที่ 6 — Validity (แวลิดิตี้) — "ความถูกต้องตามรูปแบบ"

**ความหมาย:** ข้อมูลอยู่ในรูปแบบ (Format) และช่วงค่า (Range) ที่กำหนดไว้หรือไม่?

> **เปรียบเทียบ:** หมายเลขโทรศัพท์ต้องมี 10 หลักขึ้นต้นด้วย 0 — ถ้าใส่ "abc" มาแทน = ไม่ถูกรูปแบบ

**ตัวอย่างจากข้อมูลอุบัติเหตุ:**
- ✅ ถูกรูปแบบ: `วันที่เกิดเหตุ = "2568-01-09"` → รูปแบบ YYYY-MM-DD ถูกต้อง
- ❌ ผิดรูปแบบ: `วันที่เกิดเหตุ = "9/1/2568"` → รูปแบบไม่สม่ำเสมอ ปนกันอยู่
- ❌ ผิดช่วงค่า: `เดือน = 13` → ไม่มีเดือนที่ 13 ในปฏิทิน
- ❌ ผิดช่วงค่า: `ระดับความรุนแรง_1_5 = 7` → กำหนดไว้แค่ 1–5

## ⚠️ ปัญหาคุณภาพข้อมูลที่พบบ่อย 4 ประเภท

---

### ปัญหาที่ 1 — ค่าว่าง (NULL Values / นัล แวลิวส์)

**NULL** (นัล) หมายถึง "ไม่มีค่า" หรือ "ไม่ทราบ" — ต่างจาก 0 หรือ ""

```
ตัวอย่างที่เห็นภาพ:
แบบสำรวจถามว่า "รายได้ต่อเดือน" → บางคนไม่กรอก → ช่องนั้น = NULL
0 = รายได้ศูนย์บาท (คนว่างงาน)
NULL = ไม่ได้ตอบคำถามนี้
```

**ผลกระทบ:**
- `AVG(เงินเดือน)` → ถ้า NULL ไม่ถูกนับ ค่าเฉลี่ยจะผิด
- `COUNT(*)` vs `COUNT(เงินเดือน)` → ได้ตัวเลขต่างกัน!

---

### ปัญหาที่ 2 — ข้อมูลซ้ำ (Duplicate Data / ดิวพลิเคท เดต้า)

```
ตัวอย่างที่เห็นภาพ:
ระบบลงทะเบียนนักเรียน กดส่งแบบฟอร์ม 2 ครั้ง → นักเรียนคนเดียวโผล่ 2 แถว
→ นับจำนวนนักเรียนได้ 501 คน ทั้งที่จริงมีแค่ 500 คน
```

**รูปแบบที่พบ:**
- ซ้ำทุกฟิลด์: กดบันทึกสองครั้ง
- ซ้ำบางฟิลด์: "กรุงเทพ" กับ "กรุงเทพฯ" กับ "Bangkok" → จังหวัดเดียวกัน 3 ชื่อ

---

### ปัญหาที่ 3 — ค่าผิดปกติ (Outlier / เอาท์ไลเออร์)

**Outlier** (เอาท์ไลเออร์) คือค่าที่ห่างไกลจากกลุ่มมากผิดปกติ

```
ตัวอย่างที่เห็นภาพ:
คะแนนสอบในชั้นเรียน: 65, 70, 72, 68, 75, 71, 69 ... และ 1 คนได้ 2 คะแนน
→ 2 คะแนนนี้อาจเป็น: พิมพ์ผิด? ขาดสอบ? หรือเกิดขึ้นจริง?
```

**จากข้อมูลอุบัติเหตุ:**
- ส่วนใหญ่ขับ 40–120 กม./ชม. → แต่มี record หนึ่งบันทึก 500 กม./ชม. = Outlier
- วันนอนโรงพยาบาล ส่วนใหญ่ 0–30 วัน → แต่มี 365 วัน = ผิดปกติ ต้องตรวจสอบ

---

### ปัญหาที่ 4 — รูปแบบไม่สม่ำเสมอ (Inconsistent Format / อินคอนซิสเทนท์ ฟอร์แมท)

```
ตัวอย่างที่เห็นภาพ:
ช่องจังหวัดในฐานข้อมูลเดียวกัน:
  แถวที่ 1: "เชียงใหม่"
  แถวที่ 2: "Chiang Mai"
  แถวที่ 3: "chiangmai"
  แถวที่ 4: "เชียงใหม่ "  ← มีช่องว่างตามท้าย!
→ ระบบมองเป็น 4 จังหวัดต่างกัน ทั้งที่คือจังหวัดเดียวกัน
```

---

## 💥 ผลกระทบของข้อมูลคุณภาพต่ำ — ตัวอย่างจริง

| สถานการณ์ | ข้อมูลผิดพลาด | ผลกระทบ |
|-----------|-------------|---------|
| **ธนาคาร** | ยอดเงินในบัญชีอัปเดตผิด | โอนเงินเกิน/ไม่ได้รับเงิน |
| **โรงพยาบาล** | น้ำหนักผู้ป่วยบันทึกผิด | ได้รับยาผิดขนาด อันตรายถึงชีวิต |
| **กรมขนส่ง** | ข้อมูลอุบัติเหตุซ้ำ | วิเคราะห์จุดเสี่ยงผิด ลงทุนซ่อมถนนผิดสาย |
| **ร้านค้าออนไลน์** | สินค้าคงเหลือติดลบ | ขายของที่ไม่มีในสต็อก ลูกค้าไม่ได้รับสินค้า |
| **งานวิจัย** | กลุ่มตัวอย่างข้อมูลซ้ำ | บทสรุปงานวิจัยผิดพลาดทั้งหมด |

# บทที่ 3.2 — การตรวจสอบคุณภาพด้วย SQL
**(3 ชั่วโมง)**

---

## 🔍 ภาพรวมกระบวนการตรวจสอบ

```
ขั้นตอนการตรวจสอบคุณภาพข้อมูลด้วย SQL:

ขั้น 1: สำรวจข้อมูลเบื้องต้น (Overview)
    ↓
ขั้น 2: หาค่าว่าง (NULL Check)
    ↓
ขั้น 3: หาข้อมูลซ้ำ (Duplicate Check)
    ↓
ขั้น 4: ตรวจสอบรูปแบบและช่วงค่า (Format & Range Check)
    ↓
ขั้น 5: ตรวจสอบความสอดคล้อง (Consistency Check)
    ↓
ขั้น 6: สรุปผลและบันทึก
```

---

## 📌 ขั้น 1: สำรวจข้อมูลเบื้องต้น

### นับจำนวน Records ทั้งหมด

```sql
-- นับจำนวนแถวทั้งหมด
SELECT COUNT(*) AS จำนวน_records_ทั้งหมด
FROM road_accidents;
-- ผลลัพธ์ที่คาดหวัง: 20000
```

### ดูตัวอย่างข้อมูล 5 แถวแรก

```sql
-- ดูหน้าตาข้อมูล
SELECT accident_id, วันที่เกิดเหตุ, จังหวัด, ความรุนแรง, อายุ_ปี
FROM road_accidents
LIMIT 5;
```

**ผลลัพธ์ตัวอย่าง:**

| accident_id | วันที่เกิดเหตุ | จังหวัด | ความรุนแรง | อายุ_ปี |
|-------------|--------------|---------|------------|--------|
| ACC2568000001 | 2568-01-09 | สมุทรสาคร | ไม่บาดเจ็บ | 24 |

## 📌 ขั้น 2: หาค่าว่าง (NULL Check)

### วิธีที่ 1 — ตรวจสอบทีละคอลัมน์

```sql
-- นับ NULL ในคอลัมน์ที่สำคัญ
SELECT
    COUNT(*) AS ทั้งหมด,
    COUNT(accident_id)          AS มี_accident_id,
    COUNT(*) - COUNT(accident_id) AS NULL_accident_id,

    COUNT(วันที่เกิดเหตุ)       AS มี_วันที่,
    COUNT(*) - COUNT(วันที่เกิดเหตุ) AS NULL_วันที่,

    COUNT(จังหวัด)              AS มี_จังหวัด,
    COUNT(*) - COUNT(จังหวัด)    AS NULL_จังหวัด,

    COUNT(อายุ_ปี)              AS มี_อายุ,
    COUNT(*) - COUNT(อายุ_ปี)   AS NULL_อายุ
FROM road_accidents;
```

> **หลักการ:** `COUNT(column)` นับเฉพาะแถวที่มีค่า (ไม่นับ NULL)
> ดังนั้น `COUNT(*) - COUNT(column)` = จำนวน NULL

### วิธีที่ 2 — กรองเอาเฉพาะแถวที่มีค่าว่าง

```sql
-- ดูแถวที่ไม่มีข้อมูลอายุ
SELECT accident_id, วันที่เกิดเหตุ, จังหวัด, อายุ_ปี
FROM road_accidents
WHERE อายุ_ปี IS NULL;
```

> **⚠️ ข้อควรจำ:** ห้ามใช้ `= NULL` เด็ดขาด! ต้องใช้ `IS NULL` เท่านั้น
> เพราะ NULL ไม่เท่ากับอะไรทั้งสิ้น แม้แต่ NULL เองก็ยังไม่เท่ากับ NULL

```sql
-- ผิด! ❌ วิธีนี้ไม่ทำงาน
WHERE อายุ_ปี = NULL

-- ถูก! ✅ วิธีนี้ถูกต้อง
WHERE อายุ_ปี IS NULL

-- ค้นหาแถวที่มีค่า (ไม่ว่าง)
WHERE อายุ_ปี IS NOT NULL
```

### วิธีที่ 3 — คำนวณเปอร์เซ็นต์ค่าว่าง

```sql
-- แสดงเปอร์เซ็นต์ NULL ของแต่ละคอลัมน์สำคัญ
SELECT
    'อายุ_ปี' AS คอลัมน์,
    COUNT(*) AS ทั้งหมด,
    SUM(CASE WHEN อายุ_ปี IS NULL THEN 1 ELSE 0 END) AS จำนวน_NULL,
    ROUND(
        SUM(CASE WHEN อายุ_ปี IS NULL THEN 1 ELSE 0 END) * 100.0 / COUNT(*),
        2
    ) AS เปอร์เซ็นต์_NULL
FROM road_accidents

UNION ALL

SELECT
    'จังหวัด' AS คอลัมน์,
    COUNT(*),
    SUM(CASE WHEN จังหวัด IS NULL THEN 1 ELSE 0 END),
    ROUND(SUM(CASE WHEN จังหวัด IS NULL THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2)
FROM road_accidents;
```

**ผลลัพธ์ตัวอย่าง:**

| คอลัมน์ | ทั้งหมด | จำนวน_NULL | เปอร์เซ็นต์_NULL |
|---------|---------|-----------|----------------|
| อายุ_ปี | 20000 | 0 | 0.00 |
| จังหวัด | 20000 | 0 | 0.00 |

## 📌 ขั้น 3: หาข้อมูลซ้ำ (Duplicate Check)

### วิธีที่ 1 — เปรียบเทียบจำนวน Total vs Distinct

```sql
-- ถ้าสองตัวนี้ต่างกัน = มีข้อมูลซ้ำ!
SELECT
    COUNT(*) AS จำนวนทั้งหมด,
    COUNT(DISTINCT accident_id) AS จำนวน_unique,
    COUNT(*) - COUNT(DISTINCT accident_id) AS จำนวนที่ซ้ำ
FROM road_accidents;
```

### วิธีที่ 2 — หา ID ที่ซ้ำกัน

```sql
-- แสดง accident_id ที่ปรากฏมากกว่า 1 ครั้ง
SELECT accident_id, COUNT(*) AS จำนวนครั้ง
FROM road_accidents
GROUP BY accident_id
HAVING COUNT(*) > 1
ORDER BY จำนวนครั้ง DESC;
```

> **อธิบาย `HAVING`:** ใช้คู่กับ `GROUP BY` เพื่อกรองผลลัพธ์หลังจากสรุปกลุ่มแล้ว
> เหมือน WHERE แต่ใช้กับ Aggregate Function (COUNT, SUM, AVG, ...)

### วิธีที่ 3 — หาข้อมูลซ้ำในทุกฟิลด์สำคัญ

```sql
-- แถวที่ทุกอย่างเหมือนกันหมด (อาจเป็นการบันทึกซ้ำ)
SELECT วันที่เกิดเหตุ, เวลาเกิดเหตุ, จังหวัด, อำเภอ,
       ประเภทยานพาหนะหลัก, อายุ_ปี,
       COUNT(*) AS จำนวนซ้ำ
FROM road_accidents
GROUP BY วันที่เกิดเหตุ, เวลาเกิดเหตุ, จังหวัด, อำเภอ,
         ประเภทยานพาหนะหลัก, อายุ_ปี
HAVING COUNT(*) > 1;
```

### วิธีที่ 4 — ตรวจค่าที่ควรเหมือนกันแต่เขียนต่างกัน

```sql
-- ดูค่าทั้งหมดในคอลัมน์จังหวัด (เพื่อหาการสะกดผิด)
SELECT จังหวัด, COUNT(*) AS จำนวน
FROM road_accidents
GROUP BY จังหวัด
ORDER BY จังหวัด;

-- ตรวจหาจังหวัดที่มีช่องว่างนำหน้าหรือท้าย
SELECT จังหวัด, LENGTH(จังหวัด) AS ความยาว
FROM road_accidents
WHERE จังหวัด != TRIM(จังหวัด);
-- ถ้ามีผลลัพธ์ = มีช่องว่างซ่อนอยู่!
```

## 📌 ขั้น 5: ตรวจสอบความสอดคล้อง (Consistency Check)

```sql
-- ตรวจสอบความขัดแย้ง: เกินความเร็วแต่ความเร็วน้อยกว่าจำกัด
SELECT accident_id,
       ความเร็วโดยประมาณ_กมชม,
       จำกัดความเร็ว_กมชม,
       เกินความเร็ว_ใช่ไม่ใช่
FROM road_accidents
WHERE เกินความเร็ว_ใช่ไม่ใช่ = 'ใช่'
  AND ความเร็วโดยประมาณ_กมชม <= จำกัดความเร็ว_กมชม;
-- ถ้ามีผลลัพธ์ = ข้อมูลขัดแย้งกันเอง!

-- ตรวจสอบ: ตายแต่บอกว่าไม่บาดเจ็บ
SELECT accident_id, จำนวนผู้เสียชีวิต, ความรุนแรง
FROM road_accidents
WHERE จำนวนผู้เสียชีวิต > 0
  AND ความรุนแรง = 'ไม่บาดเจ็บ';

-- ตรวจสอบ: ผลรวมผู้บาดเจ็บ+เสียชีวิตมากกว่าจำนวนคนในรถ
SELECT accident_id,
       จำนวนยานพาหนะที่เกี่ยวข้อง,
       จำนวนผู้เสียชีวิต,
       จำนวนผู้บาดเจ็บ,
       จำนวนผู้เสียชีวิต + จำนวนผู้บาดเจ็บ AS รวมผู้ประสบเหตุ
FROM road_accidents
WHERE จำนวนผู้เสียชีวิต + จำนวนผู้บาดเจ็บ > 10;
-- ถ้า > 10 คน อาจผิดปกติ (รถ 1-2 คันไม่น่ามีผู้โดยสารมากขนาดนั้น)
```

#### ตัวเลือก 1 — ลบแถวที่มี NULL (`dropna`)

```python
# ลบแถวที่มี NULL ในคอลัมน์สำคัญ
```

#### ตัวเลือก 2 — เติมค่าแทน NULL (`fillna`)

```python
# เติมค่าเฉลี่ยแทน NULL ในคอลัมน์ตัวเลข
df['อายุ_ปี'] = df['อายุ_ปี'].fillna(df['อายุ_ปี'].median())
# ใช้ median (มัธยฐาน) ดีกว่า mean เพราะทนต่อ Outlier

# เติมค่า "ไม่ทราบ" แทน NULL ในคอลัมน์ข้อความ
df['หมายเลขถนน'] = df['หมายเลขถนน'].fillna('ไม่มีหมายเลข')

# เติมค่า 0 แทน NULL ในค่าใช้จ่าย
df['ค่าใช้จ่ายรวม_บาท'] = df['ค่าใช้จ่ายรวม_บาท'].fillna(0)
```

> **เปรียบเทียบ Mean vs Median:**
> ```
> ค่าใช้จ่าย: 1,000 / 2,000 / 1,500 / 3,000 / 500,000 บาท
> Mean (ค่าเฉลี่ย)  = (1000+2000+1500+3000+500000)/5 = 101,500  ← ถูกบิดโดย Outlier!
> Median (มัธยฐาน) = 2,000  ← ค่ากลางจริง ๆ ของกลุ่ม
> ```

```python
# วิธีที่ 1 — ตัดตาม Business Rule (กฎที่รู้แน่นอน)
# อายุผู้ขับขี่ต้องอยู่ระหว่าง 15–90 ปี
df_valid = df[(df['อายุ_ปี'] >= 15) & (df['อายุ_ปี'] <= 90)]

Q1 = df['ความเร็วโดยประมาณ_กมชม'].quantile(0.25)  # ค่าที่ 25%
Q3 = df['ความเร็วโดยประมาณ_กมชม'].quantile(0.75)  # ค่าที่ 75%
IQR = Q3 - Q1  # ช่วงกลาง 50%

# ขอบบน/ล่างที่ยอมรับ
lower_bound = Q1 - 1.5 * IQR
upper_bound = Q3 + 1.5 * IQR

print(f"ยอมรับความเร็ว: {lower_bound:.0f} – {upper_bound:.0f} กม./ชม.")

# กรองเอาเฉพาะที่อยู่ในช่วงปกติ
df_no_outlier = df[
    (df['ความเร็วโดยประมาณ_กมชม'] >= lower_bound) &
    (df['ความเร็วโดยประมาณ_กมชม'] <= upper_bound)
]
```

### 🔹 สูตร Pipeline ทำความสะอาดแบบสมบูรณ์

```python
import pandas as pd

# ============================
# STEP 1: โหลดข้อมูล
# ============================
df = pd.read_csv('thailand_road_accidents_2568.csv', encoding='utf-8-sig')
print(f"✅ โหลดสำเร็จ: {df.shape[0]} แถว, {df.shape[1]} คอลัมน์")

# ============================
# STEP 2: ตรวจสอบ NULL
# ============================
null_before = df.isnull().sum().sum()
print(f"⚠️  NULL ทั้งหมด: {null_before} ค่า")

# เติมค่า NULL
df['อายุ_ปี'] = df['อายุ_ปี'].fillna(df['อายุ_ปี'].median())
df['หมายเลขถนน'] = df['หมายเลขถนน'].fillna('ไม่มีหมายเลข')

# ============================
# STEP 3: ลบข้อมูลซ้ำ
# ============================
dup_count = df.duplicated(subset=['accident_id']).sum()
df = df.drop_duplicates(subset=['accident_id'])
print(f"🗑️  ลบซ้ำ: {dup_count} แถว")

# ============================
# STEP 4: Standardize ข้อความ
# ============================
text_cols = ['จังหวัด', 'ภูมิภาค', 'ประเภทยานพาหนะหลัก', 'ความรุนแรง']
for col in text_cols:
    df[col] = df[col].str.strip()

# ============================
# STEP 5: แปลงประเภทข้อมูล
# ============================
df['วันที่เกิดเหตุ'] = pd.to_datetime(df['วันที่เกิดเหตุ'])
df['อายุ_ปี'] = df['อายุ_ปี'].astype(float)

# ============================
# STEP 6: กรอง Outlier
# ============================
df_clean = df[
    (df['อายุ_ปี'].between(5, 100)) &
    (df['ความเร็วโดยประมาณ_กมชม'].between(10, 200)) &
    (df['เดือน'].between(1, 12))
]

outlier_removed = len(df) - len(df_clean)
print(f"📊 กรอง Outlier: {outlier_removed} แถว")

# ============================
# STEP 7: บันทึกไฟล์ใหม่
# ============================
df_clean.to_csv('road_accidents_cleaned.csv', index=False, encoding='utf-8-sig')
print(f"💾 บันทึกสำเร็จ: road_accidents_cleaned.csv ({len(df_clean)} แถว)")
```

## 🐍 สร้าง Data Quality Report อัตโนมัติด้วย Python

```python
import pandas as pd
from datetime import datetime

def generate_data_quality_report(filepath, output_file='dq_report.html'):
    """
    สร้าง Data Quality Report อัตโนมัติ
    """

    # โหลดข้อมูล
    df = pd.read_csv(filepath, encoding='utf-8-sig')

    # ============================
    # รวบรวมข้อมูลสำหรับ Report
    # ============================

    # 1. ข้อมูลเบื้องต้น
    total_rows = len(df)
    total_cols = len(df.columns)

    # 2. สรุป NULL
    null_counts = df.isnull().sum()
    null_pct = (null_counts / total_rows * 100).round(2)

    # 3. ข้อมูลซ้ำ
    dup_count = df.duplicated().sum()

    # 4. สถิติตัวเลข
    numeric_stats = df.describe()

    # ============================
    # สร้าง HTML Report
    # ============================
    report_date = datetime.now().strftime("%d/%m/%Y %H:%M")

    html = f"""
<!DOCTYPE html>
<html lang="th">
<head>
    <meta charset="UTF-8">
    <title>Data Quality Report</title>
    <style>
        body {{ font-family: 'Sarabun', sans-serif; margin: 30px; background: #f5f7fa; }}
        h1 {{ color: #2c3e50; border-bottom: 3px solid #3498db; padding-bottom: 10px; }}
        h2 {{ color: #2980b9; margin-top: 30px; }}
        table {{ border-collapse: collapse; width: 100%; background: white;
                 box-shadow: 0 2px 5px rgba(0,0,0,0.1); margin: 15px 0; }}
        th {{ background: #3498db; color: white; padding: 10px 15px; text-align: left; }}
        td {{ padding: 8px 15px; border-bottom: 1px solid #ecf0f1; }}
        tr:hover {{ background: #f0f8ff; }}
        .ok {{ color: green; font-weight: bold; }}
        .warning {{ color: orange; font-weight: bold; }}
        .error {{ color: red; font-weight: bold; }}
        .summary-box {{ background: white; padding: 20px; border-radius: 8px;
                        box-shadow: 0 2px 5px rgba(0,0,0,0.1); display: inline-block;
                        margin: 10px; min-width: 150px; text-align: center; }}
        .summary-number {{ font-size: 2em; font-weight: bold; color: #2980b9; }}
    </style>
</head>
<body>
    <h1>📊 รายงานคุณภาพข้อมูล (Data Quality Report)</h1>
    <p><strong>ไฟล์:</strong> {filepath} |
       <strong>วันที่ตรวจ:</strong> {report_date}</p>

    <h2>1. ข้อมูลเบื้องต้น</h2>
    <div>
        <div class="summary-box">
            <div class="summary-number">{total_rows:,}</div>
            <div>จำนวน Records</div>
        </div>
        <div class="summary-box">
            <div class="summary-number">{total_cols}</div>
            <div>จำนวน Columns</div>
        </div>
        <div class="summary-box">
            <div class="summary-number {'<span class=\\"error\\">' if dup_count > 0 else ''}{dup_count}{'</span>' if dup_count > 0 else ''}</div>
            <div>แถวซ้ำ</div>
        </div>
        <div class="summary-box">
            <div class="summary-number">{null_counts.sum():,}</div>
            <div>ค่า NULL ทั้งหมด</div>
        </div>
    </div>

    <h2>2. ตรวจสอบ NULL แต่ละคอลัมน์</h2>
    <table>
        <tr><th>คอลัมน์</th><th>ข้อมูลที่มี</th><th>NULL</th><th>%NULL</th><th>สถานะ</th></tr>
"""

    for col in df.columns:
        n_null = null_counts[col]
        pct = null_pct[col]
        status = '✅ ปกติ' if pct == 0 else ('⚠️ ควรตรวจสอบ' if pct < 5 else '❌ มีปัญหา')
        css = 'ok' if pct == 0 else ('warning' if pct < 5 else 'error')
        html += f"""
        <tr>
            <td>{col}</td>
            <td>{total_rows - n_null:,}</td>
            <td>{n_null:,}</td>
            <td>{pct}%</td>
            <td class="{css}">{status}</td>
        </tr>"""

    html += """
    </table>

    <h2>3. สถิติข้อมูลตัวเลข</h2>
    <p>ตรวจสอบค่า min/max เพื่อหา Outlier ที่น่าสงสัย</p>
"""
    html += numeric_stats.to_html(classes='', border=0, float_format='{:,.2f}'.format)

    html += """
    <h2>4. สรุปผลและข้อเสนอแนะ</h2>
    <table>
        <tr><th>มิติคุณภาพ</th><th>ผล</th><th>ข้อเสนอแนะ</th></tr>
        <tr><td>Completeness (ความสมบูรณ์)</td>
            <td>ดูคอลัมน์ที่มี NULL > 0%</td>
            <td>fillna() หรือ dropna()</td></tr>
        <tr><td>Uniqueness (ไม่ซ้ำ)</td>
            <td>ดูจำนวนแถวซ้ำ</td>
            <td>drop_duplicates()</td></tr>
        <tr><td>Accuracy (ถูกต้อง)</td>
            <td>ดูค่า min/max ใน describe()</td>
            <td>กรองค่าที่เป็นไปไม่ได้</td></tr>
    </table>

</body>
</html>
"""

    # บันทึกไฟล์
    with open(output_file, 'w', encoding='utf-8') as f:
        f.write(html)

    print(f"✅ สร้างรายงานสำเร็จ: {output_file}")
    return df

# ใช้งาน
df = generate_data_quality_report('thailand_road_accidents_2568.csv', 'dq_report.html')
```

```
หน่วยที่ 3 — คุณภาพข้อมูลและการตรวจสอบ

   Outlier   → กรองตาม business rule หรือ IQR method
```
